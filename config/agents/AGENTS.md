## Devenv

- Keep in mind that you are in NixOS environment

## Code style

- Write code that is clean, simple and easy to understand and reason about.
- Avoid side-effects: every function must do one thing obvious from the name with no side effects.
- Avoid comments if they are not vital. Code should explain itself.
- Always be consistent with the codebase: check similar existing code and follow it's patterns.
- Avoid workarounds or code smell. When task requires the workaround, let me know.
- Always read `PLAN.md`, `SPEC.md`, and `ADR.md` from the repository root if they are present.
- Use language mechanisms (tests, asserts, typing system) to ensure correct behaviour when possible.
- Clarify with me when anything is unclear

## Behaviour

- Be mindful about the fact that human is editing the code too, so you need to be careful not to overwrite their changes.
- Try to minimize the amount of code. Always try to simplify and make your code clever and effective.
- You are allowed to use local git commands but do not push anything. Make sure to commit every atomic change with clear commit message.
- Don't manually write the code that should be generated by tools. Instruct user about the generation steps.
- Always write tests that assert the business requirements. Don't write tests that confirm implementation details, regression tests or are too specific.
- If you need regression test for self-verification you can write it but remove it afterwards.
- Keep tests minimal, clean and easy to understand and reason about. Avoid writing tests that are not really necessary. Always try to minimize test code before you finish.
- Never adjust tests for happy paths to ensure pass.

## Attention

- Don't pay attention to unrelated untracked files in the git tree.
- Ignore PLAN.md when it obviously has nothing to do with a task.
